一句話版：Prompt 是「這次請你幫我做這件事」的即時指令；Skill 是「把一套做法打包好，之後每次都照這套流程做」的能力包，核心通常是一份 `SKILL.md`。

你可以把 Prompt 想成便利貼。今天要 AI 幫你改一封信，就寫一張便利貼：「請幫我把這封信改得禮貌一點」。任務做完，便利貼大多就結束了。

Skill 比較像食譜卡或工具箱。你不只寫「幫我做菜」，而是把材料、步驟、口味、檢查方式都整理好。下次遇到同類任務，不用重新想一遍，直接拿出同一張卡照著做。

## 同一個任務怎麼看

假設你要翻譯一本書。

只用 Prompt 時，你可能每次都要重新貼一長串規則：用台灣慣用語、保留 Markdown、不要翻譯專有名詞、先拆章節、翻完後檢查語氣一致。第一章貼一次，第二章又貼一次；過幾天回來繼續翻，還要回想當初有哪些規則。

做成 Skill 時，你會把這些規則整理進 `SKILL.md`。裡面可以寫清楚：什麼時候啟用、輸入需要什麼、每章怎麼處理、遇到術語怎麼建立詞彙表、最後怎麼檢查。之後每一本書都沿用同一套流程，輸出比較穩定，也比較容易修正。

## SKILL.md 扮演什麼角色

`SKILL.md` 是 Skill 的說明書，也是 AI 執行這套能力時最先看的入口。它通常會包含三類資訊：

- 什麼時候應該使用這個 Skill。
- 任務要怎麼拆步驟、用什麼格式輸出。
- 有哪些注意事項、參考檔案、工具或腳本需要搭配。

所以，Skill 不只是把一段 Prompt 存起來。它更像把「會做這件事的人腦中的流程」寫成一份可重複使用的操作手冊。

## 什麼時候用 Prompt

如果你的任務符合下面情況，用 Prompt 通常就夠了：

- 只會做一次。
- 結果不需要和之後的輸出保持一致。
- 規則很短，貼一次就說得清楚。
- 你還在摸索想要的方向，暫時不確定流程。

例子：請 AI 幫你想 10 個活動名稱、改一封今天要寄出的信、快速解釋一個名詞。

## 什麼時候做成 Skill

如果你的任務符合下面情況，就值得整理成 Skill：

- 會重複做很多次。
- 每次都希望格式、語氣、檢查方式一致。
- 規則太長，每次重貼很麻煩。
- 你想把流程分享給別人，或未來持續改版。
- 任務需要參考檔案、範例、工具、API key 或腳本。

例子：每週產出固定格式的研究摘要、替不同職缺客製履歷、把論文分析成簡報和重點圖、幫小朋友生成固定語氣的睡前故事。

## 最簡單的判斷法

臨時、一次性、規則短：用 Prompt。

重複、要一致、會維護：做成 Skill。

如果你不確定，先用 Prompt 試跑幾次。當你發現自己一直複製同一段規則、一直提醒 AI 同一件事，那就是把它整理成 Skill 的時候。
